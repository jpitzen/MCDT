## Priority 1 — Close Critical DG Gaps (Q1 2026)

### P1.1: S3 CSI Driver Completion (Phase 5)
**Gap**: S3 bucket exists but Mountpoint CSI driver addon not installed
**Work**:
- Add `aws_eks_addon.s3_csi` to `terraform/modules/kubernetes_cluster/main.tf` (pattern: copy EBS/EFS addon blocks)
- Add S3 CSI IRSA role to `terraform/modules/kubernetes_cluster/iam.tf`
- Add S3 StorageClass to `terraform/modules/kubernetes_cluster/storage.tf`
**Effort**: 1 day
**Coverage after**: ~90%

### P1.2: ALB Controller & Ingress (Phase 9)
**Gap**: No AWS Load Balancer Controller, no Ingress resource generation
**Work**:
- Add ALB Controller IAM policy + IRSA role to `iam.tf` (DG Phase 0 Step 6)
- Add Helm release for `aws-load-balancer-controller` to a new `terraform/modules/kubernetes_cluster/helm.tf` or integrate via `kubernetesManifestGenerator`
- Wire `kubernetesManifestGenerator.js` Ingress templates into deployment flow with ALB annotations, path routing, SSL cert ARN
- Frontend: add Ingress configuration to Phase5Deploy or Phase6Operations
**Effort**: 3 days
**Coverage after**: ~80%

### P1.3: Cluster Autoscaler Deployment (Phase 10)
**Gap**: IAM + ASG tags exist, but autoscaler pod is never deployed
**Work**:
- Add Cluster Autoscaler Helm chart or K8s Deployment manifest to terraform flow
- `kubernetesManifestGenerator.js` already has HPA support — wire autoscaler deployment into Phase6Operations
**Effort**: 1 day
**Coverage after**: ~95%

### P1.4: Database Schema Deployment (Phase 6)
**Gap**: RDS created but no app DB user, no K8s ConfigMap/Secret, no schema deployment
**Work**:
- Extend `databaseScriptExecutor.js` to run during deployment lifecycle (post-RDS provisioning)

**Effort**: 2 days
**Coverage after**: ~85%

## Sanity Check Residual Items

From [sc20260302-1010](../sanitychecks/sc20260302-1010.md) — all F1–F8 findings were fixed in `copilot-instructions.md`. One advisory remains:

- **F9** (no fix needed): Auth applied at router level is the convention, but `auth.js` routes naturally skip it. Documented as-is; correct as a pattern guide.

### Dual migration system (F4) — architectural decision needed:
Two migration systems coexist (`sequelize-cli` + raw SQL `run-migrations.js`). This is now documented but should be unified long-term. Recommend standardizing on one approach:
- **Option A**: Sequelize CLI for all (tooling support, up/down migrations)
- **Option B**: Raw SQL for all (simpler, idempotent, no ORM dependency)
