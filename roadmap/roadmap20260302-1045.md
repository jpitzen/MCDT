## Executive Directive

> *"Our manual process endstate will be the endstate for the automated process. The external access will be the only difference."*

This means: after the automated platform deploys, the running Kubernetes state **must be identical** to the manually-deployed DG03 endstate. The only variable is whether the deployment uses **internal-only access** (ClusterIP + internal hostname) or **external access** (LoadBalancer + SSL + public DNS).

| Resource | Status | Reason |
|----------|--------|--------|
| PVC `zlservertemp-efs` | Unused | zlserver uses `emptyDir` instead |
| PVC `zltikatemp-efs` | Unused | zlui uses `emptyDir` named `zltikatemp` |

## § C — Access Mode Design: Internal-Only vs External

### C.1 Architecture Comparison

```
INTERNAL-ONLY                           EXTERNAL ACCESS
─────────────────                       ─────────────────
┌─────────────┐                        ┌─────────────┐
│  K8s Cluster │                        │  K8s Cluster │
│              │                        │              │
│  zlui ──────────► ClusterIP           │  zlui ──────────► LoadBalancer ──► ELB
│  (port 80)   │    (8080→80)           │  (port 80)   │    (8080→80)     │
│              │         │              │              │                   │
│              │    kubectl port-fwd    │              │         ┌─────────┘
│              │    or VPN/bastion      │              │         │
└─────────────┘                        └─────────────┘         ▼
      │                                                   DNS CNAME
      ▼                                                   (zlpsaws.zlpsonline.com)
  Admin-only access                                            │
  via internal tools                                      SSL Certificate
                                                          (ACM or uploaded)
```

### C.2 What Changes Between Modes

Only **4 artifacts differ** between internal and external access. Everything else is identical:

| Artifact | Internal-Only | External Access |
|----------|:-------------|:---------------|
| `pmappurl-config` → `HAS_SSL` | `false` | `true` |
| `pmappurl-config` → `web.server.URL` | `zlui-service.<ns>.svc.cluster.local` (or blank) | `zlpsaws.zlpsonline.com` (user-provided domain) |
| `tomcat-server-config` → `proxyName` | *(empty or localhost)* | `zlpsaws.zlpsonline.com` |
| `tomcat-server-config` → `scheme`/`secure`/`proxyPort` | `http` / `false` / `8080` | `https` / `true` / `443` |
| `zlui-service` type | `ClusterIP` | `LoadBalancer` |
| `zlui-ssl-cert` Secret | *(not created)* | TLS cert + key |
| DNS CNAME record | *(not created)* | Domain → ELB hostname |

### C.4 Backend Support — Access Mode in Deployment Flow

**New field on `deployments` table**: `access_mode` ENUM('internal', 'external') DEFAULT 'internal'

**New fields on `deployments` table** (nullable, used when `access_mode = 'external'`):
- `external_domain` VARCHAR(255)
- `ssl_mode` ENUM('acm', 'upload')
- `ssl_cert_arn` VARCHAR(255)

**Manifest generation branching**: When the manifest generator produces ConfigMaps and Services, it checks `deployment.access_mode`:

```
if access_mode == 'internal':
    pmappurl-config:  HAS_SSL=false, web.server.URL=(blank)
    tomcat-server:    scheme=http, secure=false, proxyName=(blank), proxyPort=8080
    zlui-service:     type=ClusterIP
    SKIP:             zlui-ssl-cert Secret, DNS record

if access_mode == 'external':
    pmappurl-config:  HAS_SSL=true, web.server.URL=${external_domain}
    tomcat-server:    scheme=https, secure=true, proxyName=${external_domain}, proxyPort=443
    zlui-service:     type=LoadBalancer, annotations for SSL
    CREATE:           zlui-ssl-cert Secret (if upload mode)
    CREATE:           Route53 CNAME → ELB hostname (post-apply)
```

### C.5 Post-Deploy Access Information

After deployment completes, Phase 6 (Operations) should display:

```

URL:    http://localhost:8080/ps
Note:   Accessible only via kubectl, VPN, or bastion host
```

**External mode**:
```
Access: https://zlpsaws.zlpsonline.com/ps
DNS:    CNAME zlpsaws.zlpsonline.com → a0f6d05f1-xxxx.us-east-1.elb.amazonaws.com
SSL:    Valid, terminating at LoadBalancer
Health: GET https://zlpsaws.zlpsonline.com/ps/status → 200
```
