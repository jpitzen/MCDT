EBS CSI Driver Overview

Phase 3 installs the Amazon EBS CSI Driver to enable Kubernetes to manage EBS volumes for persistent storage, which is essential for ZooKeeper's stateful data.

ℹ️ IAM Resources - Global Service

Important: IAM is a global AWS service. If you have already created these IAM policies and roles for another region or deployment:

  • Skip policy creation: Use your existing AmazonEBSCSIDriverPolicy
  • Skip role creation: Reuse existing service account roles if they have the same OIDC provider
  • Verify ARNs: Ensure policy and role ARNs match your existing resources
  • Check trust relationships: Verify the IAM role trusts your cluster's OIDC provider

To list existing policies: aws iam list-policies --query "Policies[?PolicyName=='AmazonEBSCSIDriverPolicy']"

🎯 Objectives

  • Create IAM policy for EBS CSI Driver
  • Create IAM role with OIDC trust relationship
  • Create Kubernetes service account with IAM role annotation
  • Install EBS CSI Driver addon via AWS CLI
  • Verify CSI driver pods are running
  • Create StorageClass for gp3 volumes
