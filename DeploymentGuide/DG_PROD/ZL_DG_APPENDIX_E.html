Appendix E: Monitoring & Logging Reference

Reference Updated: Dec 23, 2025

This appendix provides monitoring and logging configurations for the ZL application deployment on AWS EKS.

📋 Monitoring Categories

  • CloudWatch metrics and alarms
  • Container Insights setup
  • Application logging configurations
  • Log aggregation and analysis
  • Alerting and notification setup
  • Performance monitoring dashboards

📊 CloudWatch Container Insights

Enable Container Insights

# Enable Container Insights for EKS cluster
aws logs create-log-group --log-group-name /aws/containerinsights/zl-eks-cluster/performance --region us-east-1
aws logs create-log-group --log-group-name /aws/containerinsights/zl-eks-cluster/application --region us-east-1

# Deploy CloudWatch agent as DaemonSet
kubectl apply -f https://raw.githubusercontent.com/aws-samples/amazon-cloudwatch-container-insights/latest/k8s-deployment-manifest-templates/deployment-mode/daemonset/container-insights-monitoring/cloudwatch-agent.yaml

# Create ConfigMap for CloudWatch agent
kubectl apply -f https://raw.githubusercontent.com/aws-samples/amazon-cloudwatch-container-insights/latest/k8s-deployment-manifest-templates/deployment-mode/daemonset/container-insights-monitoring/cwagent-configmap.yaml

Container Insights Metrics

Container Insights automatically collects the following metrics:

  • Node metrics: CPU, memory, disk, network utilization
  • Pod metrics: CPU, memory usage by pod
  • Container metrics: CPU, memory usage by container
  • Cluster metrics: Node count, pod count, container count
  • Control plane metrics: API server requests, etcd metrics

📈 CloudWatch Alarms

Cluster Health Alarms

# High CPU utilization alarm
aws cloudwatch put-metric-alarm \
  --alarm-name "zl-eks-high-cpu" \
  --alarm-description "EKS cluster high CPU utilization" \
  --metric-name "node_cpu_utilization" \
  --namespace "ContainerInsights" \
  --statistic "Average" \
  --period 300 \
  --threshold 80 \
  --comparison-operator "GreaterThanThreshold" \
  --dimensions Name=ClusterName,Value=zl-eks-cluster \
  --evaluation-periods 2 \
  --alarm-actions arn:aws:sns:us-east-1::zl-alerts

# High memory utilization alarm
aws cloudwatch put-metric-alarm \
  --alarm-name "zl-eks-high-memory" \
  --alarm-description "EKS cluster high memory utilization" \
  --metric-name "node_memory_utilization" \
  --namespace "ContainerInsights" \
  --statistic "Average" \
  --period 300 \
  --threshold 85 \
  --comparison-operator "GreaterThanThreshold" \
  --dimensions Name=ClusterName,Value=zl-eks-cluster \
  --evaluation-periods 2 \
  --alarm-actions arn:aws:sns:us-east-1::zl-alerts

Application Performance Alarms

# Pod restart alarm
aws cloudwatch put-metric-alarm \
  --alarm-name "zl-app-pod-restarts" \
  --alarm-description "High pod restart rate" \
  --metric-name "pod_number_of_container_restarts" \
  --namespace "ContainerInsights" \
  --statistic "Sum" \
  --period 300 \
  --threshold 5 \
  --comparison-operator "GreaterThanThreshold" \
  --dimensions Name=ClusterName,Value=zl-eks-cluster Name=Namespace,Value=zl-app \
  --evaluation-periods 1 \
  --alarm-actions arn:aws:sns:us-east-1::zl-alerts

# ALB 5xx errors alarm
aws cloudwatch put-metric-alarm \
  --alarm-name "zl-alb-5xx-errors" \
  --alarm-description "High ALB 5xx error rate" \
  --metric-name "HTTPCode_Target_5XX_Count" \
  --namespace "AWS/ApplicationELB" \
  --statistic "Sum" \
  --period 300 \
  --threshold 10 \
  --comparison-operator "GreaterThanThreshold" \
  --dimensions Name=LoadBalancer,Value=app/zl-alb/1234567890abcdef \
  --evaluation-periods 1 \
  --alarm-actions arn:aws:sns:us-east-1::zl-alerts

📝 Application Logging

Fluent Bit Configuration

apiVersion: v1
kind: ConfigMap
metadata:
  name: fluent-bit-config
  namespace: amazon-cloudwatch
  labels:
    k8s-app: fluent-bit
data:
  fluent-bit.conf: |
    [SERVICE]
        Flush         5
        Log_Level     info
        Daemon        off

    [INPUT]
        Name              tail
        Path              /var/log/containers/*zl-app*.log
        Parser            docker
        Tag               application.*
        Refresh_Interval  5

    [OUTPUT]
        Name  cloudwatch_logs
        Match application.*
        Region us-east-1
        Log_Group_Name /aws/containerinsights/zl-eks-cluster/application
        Log_Stream_Prefix fluent-bit-
        Auto_Create_Group On

Application Log Groups

# Create log groups for different components
aws logs create-log-group --log-group-name /zl/application/server --region us-east-1
aws logs create-log-group --log-group-name /zl/application/ui --region us-east-1
aws logs create-log-group --log-group-name /zl/zookeeper/cluster --region us-east-1

# Set log retention
aws logs put-retention-policy --log-group-name /zl/application/server --retention-in-days 30 --region us-east-1
aws logs put-retention-policy --log-group-name /zl/application/ui --retention-in-days 30 --region us-east-1
aws logs put-retention-policy --log-group-name /zl/zookeeper/cluster --retention-in-days 30 --region us-east-1

🚨 Alerting & Notifications

SNS Topic for Alerts

# Create SNS topic for alerts
aws sns create-topic --name zl-alerts --region us-east-1

# Subscribe email to alerts
aws sns subscribe \
  --topic-arn arn:aws:sns:us-east-1::zl-alerts \
  --protocol email \
  --notification-endpoint admin@company.com

# Create alarm action for critical alerts
aws sns create-topic --name zl-critical-alerts --region us-east-1

Alert Escalation Policy

🔍 Log Analysis & Queries

CloudWatch Insights Queries

# Find application errors
fields @timestamp, @message
| filter @logStream like /zl-server/
| filter @message like /ERROR|Exception/
| sort @timestamp desc
| limit 100

# Monitor API response times
fields @timestamp, response_time, request_path
| filter ispresent(response_time)
| filter response_time > 5000
| sort response_time desc
| limit 50

# Track database connection issues
fields @timestamp, @message
| filter @message like /connection|timeout|database/
| sort @timestamp desc
| limit 20

Log Metric Filters

# Create metric filter for application errors
aws logs put-metric-filter \
  --log-group-name /zl/application/server \
  --filter-name "ApplicationErrors" \
  --filter-pattern "ERROR" \
  --metric-transformations \
    metricName=ApplicationErrors,metricNamespace=ZL/Application,metricValue=1

# Create metric filter for HTTP 5xx responses
aws logs put-metric-filter \
  --log-group-name /zl/application/server \
  --filter-name "HTTP5xxErrors" \
  --filter-pattern '"status": 5*' \
  --metric-transformations \
    metricName=HTTP5xxErrors,metricNamespace=ZL/Application,metricValue=1
