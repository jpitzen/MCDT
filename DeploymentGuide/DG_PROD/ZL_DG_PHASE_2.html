Phase 2: ECR & Image Management

Step 1 of 6
Overview & Repository Info

Phase 2 sets up Amazon Elastic Container Registry (ECR) repositories and manages container images for the ZL application deployment.

📘 Deployment Scenarios

This guide supports two infrastructure scenarios:

  • New Infrastructure: Follow all steps to create a new ECR repository
  • Existing Infrastructure: If your AWS account already has an ECR repository, you can use it instead of creating a new one. Simply use your existing repository name and URI in Steps 3-4.

Registry ID: The registry ID is your 12-digit AWS Account ID. Find it with: aws sts get-caller-identity --query Account --output text

📌 Version Note: This guide uses ZL Version 11.1.0 (Build 1140). Ensure your Docker images match your target ZL version before pushing to ECR. Update image tags in all Kubernetes manifests accordingly.

🎯 Objectives

  • Create ECR repositories for application images
  • Configure repository policies and lifecycle rules
  • Push container images to ECR
  • Verify image integrity and security scanning
  • Set up automated image builds and deployments
