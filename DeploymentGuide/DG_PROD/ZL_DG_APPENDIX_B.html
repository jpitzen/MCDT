Appendix B: Kubernetes Manifests Reference

Reference Updated: Dec 23, 2025

This appendix provides complete, working Kubernetes manifests from the us-east-1 deployment. All manifests have been tested and verified.

🔴 IMPORTANT: December 22, 2025 Update

These manifests reflect the actual working deployment. Key changes from previous versions:

  • K8s-optimized images with Java 21 pre-installed
  • TIKA_HOST environment variable for Tika connectivity
  • Critical Tika command override with sleep infinity
  • TrustServerCertificate=true in JDBC URL for RDS

📋 Manifest Categories

  • ConfigMaps - Application and database configuration
  • Secrets - Database credentials
  • Storage - PVCs and StorageClasses
  • Services - Service definitions
  • Deployments - Application deployments
  • ZooKeeper - StatefulSet configuration

🔐 ConfigMaps

The repository ships a consolidated file at aws/clean/configmaps.yaml that contains every ConfigMap summarized here. Applying that single file keeps your deployment and the guide in sync; use kubectl apply -f aws/clean/configmaps.yaml when you reach this section.

ZooKeeper Client Configuration

⚠️ Critical: The tcdb.cfg MUST include TrustServerCertificate=true in the JDBC URL for RDS connectivity. 
apiVersion: v1
kind: ConfigMap
metadata:
  name: zkclient-config
data:
  # zkQuorum.cfg - ZooKeeper Quorum Configuration
  zkQuorum.cfg: |
    #if !_ZK_SERVER_CONFIG_FILE_IS_INCLUDED
    #define _ZK_SERVER_CONFIG_FILE_IS_INCLUDED = true

    // ZooKeeper Authentication Key
    _zlzk.auth=#com.zlti.zlzookeeper.ZLZKAuth~~

    // ZooKeeper Quorum for Kubernetes
    zkQuorum.0=0~~zlzookeeper-0.zk-hs.default.svc.cluster.local~~2181~~2888~~3888~~/var/ZipLip/DATA/ZooKeeper/zk0
    zkQuorum.1=1~~zlzookeeper-1.zk-hs.default.svc.cluster.local~~2181~~2888~~3888~~/var/ZipLip/DATA/ZooKeeper/zk1
    zkQuorum.2=2~~zlzookeeper-2.zk-hs.default.svc.cluster.local~~2181~~2888~~3888~~/var/ZipLip/DATA/ZooKeeper/zk2
    acv.zkQuorum=#wsi.config.AllConfigVariables~~@NAMES_THAT_START_WITH@~~zkQuorum.
    #endif

  # tcdb.cfg - Database Protected Key Configuration
  tcdb.cfg: |
    // Database Configuration
    #define DB_MSSQL_DEFAULT = true

    _db.param.user = pfuser
    _db.password = 

    // CRITICAL: TrustServerCertificate=true is REQUIRED for RDS
    #if DB_MSSQL_DEFAULT
    _db.param.jdbcUrl=jdbc:sqlserver://:1433;TrustServerCertificate=true
    _db.param.DatabaseName=ZLDB
    #endif

    acv.db=#wsi.config.AllConfigVariables~~@NAMES_THAT_START_WITH@~~_db.param.
    _db.KeyName = zltc/db
    _tcdb = #com.zlti.zlzookeeper.ZKProtectedKeyFactory~~@_db.KeyName@~~@_db.password@~~@acv.db@

  # zkClient.cfg
  zkClient.cfg: |
    _zkCluster = /opt/ZipLip/ZLZooKeeper/config/zkQuorum.cfg
    __zk.main=#com.zlti.zlzookeeper.ZLZooKeeper~~@_zkCluster@~~3000~~true

  # tc.cfg - Site Configuration
  tc.cfg: |
    _cfg.machine=tc/K8S
    _cfg.param.pmApp.site.name = K8S
    _cfg.param.coord.cluster.default.name = DEFAULT
    _cfg.param.coord.cluster.initial.executors = 5
    _cfg.param.coord.cluster.service.executors = 3
    _cfg.param.coord.cluster.managed.executors = 3
    acv.mach=#wsi.config.AllConfigVariables~~@NAMES_THAT_START_WITH@~~_cfg.param.
    _tc = #com.zlti.zlzookeeper.ZKConfigFactory~~@_cfg.machine@~~@acv.mach@

🚀 Deployments

Tika Deployment

🔴 CRITICAL: The command override with sleep infinity is REQUIRED. Without it, the Tika Java process exits before the listener binds to port 9972. 
apiVersion: apps/v1
kind: Deployment
metadata:
  name: zltika
  labels:
    app: zltika
spec:
  replicas: 1
  selector:
    matchLabels:
      app: zltika
  template:
    metadata:
      labels:
        app: zltika
    spec:
      containers:
      - name: zltika
        image: .dkr.ecr.us-east-1.amazonaws.com/ue1-zlps-ecr-01:zltika11.1.0-b1140
        # CRITICAL: This command is REQUIRED - prevents container exit
        command:
          - /bin/sh
          - -c
          - cd /opt/ZipLip/ZLTikaConvertor/bin && ./zltikadiag.sh ZLTika /tmp/ZLTika & sleep infinity
        ports:
        - containerPort: 9972
        resources:
          requests:
            memory: "1Gi"
            cpu: "500m"
          limits:
            memory: "2Gi"
            cpu: "1000m"

  name: zlzookeeper
  labels:

      initContainers:

        image: busybox:1.28

        - name: datadir
          mountPath: /var/ZipLip/DATA/ZooKeeper
      containers:
      - name: zlzookeeper

        - containerPort: 2181
          name: client

          name: server
        - containerPort: 3888
          name: leader-election
        env:
        - name: ZK_REPLICAS
          value: "3"
        - name: ZK_HEAP_SIZE
          value: "512M"
        - name: ZK_TICK_TIME
          value: "2000"
        - name: ZK_INIT_LIMIT
          value: "10"
        - name: ZK_SYNC_LIMIT
          value: "5"
        - name: ZK_MAX_CLIENT_CNXNS
          value: "60"
        volumeMounts:
        - name: datadir
          mountPath: /var/ZipLip/DATA/ZooKeeper
        - name: zk-config
          mountPath: /opt/ZipLip/ZLZooKeeper/config
        resources:

📋 Deployment Order

Apply Manifests in This Order:

  1. Storage Classes & PVCs - kubectl apply -f storage.yaml
  2. Secrets - kubectl apply -f secrets.yaml
  3. ConfigMaps - kubectl apply -f configmaps.yaml
  4. Services - kubectl apply -f services.yaml
  5. ZooKeeper - kubectl apply -f zlzookeeper-statefulset.yaml
    Wait for all 3 pods to be Running before continuing
  6. Tika - kubectl apply -f zltika-deployment.yaml
    Wait for Tika to be Running and listening on port 9972
  7. ZL Server - kubectl apply -f zlserver-deployment.yaml
    Wait for "Done initializing ZLServer" in logs
  8. ZL UI - kubectl apply -f zlui-deployment.yaml
  9. ZL Search - kubectl apply -f zlsearch-deployment.yaml
