ZL Application AWS EKS Deployment Guide

Home Updated: Dec 23, 2025

🏠 Welcome to the ZL Application Deployment Guide

This comprehensive guide walks you through deploying the ZL application on Amazon EKS (Elastic Kubernetes Service) with all necessary AWS integrations including ECR, EBS, EFS, S3, and RDS.

📌 Version Information

This deployment guide is based on ZL Version 11.1.0 (Build 1140) - 11.06.2025-11:47

While the deployment steps and procedures are applicable to other builds and versions, please ensure:

  • Docker images match your target ZL version
  • Kubernetes YAML manifests reference the correct image tags
  • ConfigMaps and JSON files contain version-appropriate configurations
  • SQL scripts in the setuporder folder match your database schema version

Always verify version compatibility when adapting this guide for different ZL builds.

🎯 What You'll Deploy

  • EKS Cluster: Managed Kubernetes control plane
  • Container Registry: ECR for Docker images
  • Storage: EBS, EFS, and S3 integrations
  • Database: RDS PostgreSQL instance
  • Application: ZL web application with ZooKeeper
  • Security: Inspector scanning and monitoring

📋 Prerequisites

Before starting deployment:

  • AWS CLI v2.x+ installed and configured
  • kubectl v1.28+ for cluster management
  • eksctl v0.165+ for cluster creation
  • Docker Desktop for container operations
  • Helm v3.x for package management
  • Git for repository cloning

See Phase 0 for detailed setup instructions

🚀 Quick Start

  1. Phase 0: Configure AWS credentials and IAM
  2. Phase 1: Create EKS cluster with worker nodes
