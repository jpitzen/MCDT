## Critical Fixes Documented

### 1. Tika Container Exit Issue
**Problem:** Tika container exits before Java listener binds to port 9972
**Solution:** Override command with `sleep infinity`:
```yaml
command:
  - /bin/sh
  - -c
  - cd /opt/ZipLip/ZLTikaConvertor/bin && ./zltikadiag.sh ZLTika /tmp/ZLTika & sleep infinity
```

### 2. Database SSL Certificate
**Problem:** JDBC connection fails with SSL certificate error
**Solution:** Add `TrustServerCertificate=true` to JDBC URL in tcdb.cfg:
```
_db.param.jdbcUrl=jdbc:sqlserver://hostname:1433;TrustServerCertificate=true
```

### 3. Superadmin Login Failure
**Problem:** NullPointerException for AliasLookup after login
**Cause:** ZLUI started before ZLServer finished initializing
**Solution:** Restart ZLUI after ZLServer shows "Done initializing":
```bash
kubectl rollout restart deployment zlui
```

## Verification Commands
```bash
# Check all pods
kubectl get pods

# Verify ZooKeeper quorum
kubectl exec zlzookeeper-0 -- bash -c "cd /opt/ZipLip/ZLZooKeeper/bin && ./zkutil -q"

# Verify Tika listening
kubectl exec $(kubectl get pods -l app=zltika -o name) -- netstat -tlnp | grep 9972

# Verify ZLServer initialized
kubectl logs deployment/zlserver | grep -i "done init"

# Get LoadBalancer URL
kubectl get svc zlui-service -o jsonpath='{.status.loadBalancer.ingress[0].hostname}:{.spec.ports[0].port}'
```
