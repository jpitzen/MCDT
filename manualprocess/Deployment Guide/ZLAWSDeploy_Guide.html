About This Document

This document provides a comprehensive guide for deploying the ZL Application on Amazon Elastic Kubernetes Service (EKS). It captures all steps taken during the initial deployment, including AWS resource provisioning, Kubernetes configuration, storage setup, and application deployment.

Prerequisites

Before starting the deployment, ensure you have the following tools installed and configured:

📋 What It Does

  • Creates IAM policy for EBS CSI Driver operations
  • Enables Kubernetes to provision and manage EBS volumes

❓ Why It's Needed

The EBS CSI Driver needs permissions to create, attach, detach, and delete EBS volumes dynamically as pods request persistent storage.

  "Statement": [

      "Effect": "Allow",

Step 0.6: Create EFS CSI Driver Policy

📋 What It Does

  • Creates IAM policy for EFS CSI Driver operations
  • Enables Kubernetes to create EFS access points and mount targets
  • Required for shared storage across multiple pods (ReadWriteMany)

❓ Why It's Needed

EFS provides shared file storage for logs, vault files, and temporary data that needs to be accessed by multiple pods simultaneously.

📄 Policy Document

Create file policies/EFS-IAM-policy.json:

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "elasticfilesystem:DescribeAccessPoints",
        "elasticfilesystem:DescribeFileSystems",
        "elasticfilesystem:DescribeMountTargets",
        "elasticfilesystem:DescribeMountTargetSecurityGroups"
      ],
      "Resource": "*"
    },
    {
      "Effect": "Allow",
      "Action": ["elasticfilesystem:CreateAccessPoint"],
      "Resource": "*",
      "Condition": {
        "StringLike": {
          "aws:RequestTag/efs.csi.aws.com/cluster": "true"
        }
      }
    },
    {
      "Effect": "Allow",
      "Action": ["elasticfilesystem:DeleteAccessPoint"],
      "Resource": "*",
      "Condition": {
        "StringEquals": {
          "aws:ResourceTag/efs.csi.aws.com/cluster": "true"
        }
      }
    },
    {
      "Effect": "Allow",
      "Action": ["ec2:DescribeAvailabilityZones"],
      "Resource": "*"
    }
  ]
}

💻 Command

# Create EFS CSI Driver policy
aws iam create-policy --policy-name AmazonEFSCSIDriverPolicy --policy-document file://policies/EFS-IAM-policy.json

Step 0.9: Enable AWS Inspector for ECR Scanning

📋 What It Does

  • Enables AWS Inspector2 for ECR container image scanning
  • Automatically scans images pushed to ECR for vulnerabilities (CVEs)
  • Provides continuous monitoring of container security posture

❓ Why It's Needed

AWS Inspector provides native vulnerability scanning for ECR images. When enabled, it automatically scans all images and continuously monitors for newly discovered CVEs. This is required for Phase 12 security scanning and comparison features.

💻 Commands

# Check current Inspector status
aws inspector2 batch-get-account-status --region us-west-1

# Enable Inspector2 for ECR scanning
aws inspector2 enable --resource-types ECR --region us-west-1

Expected Output:

PHASE 5: S3 CSI Driver Optional

Home → Phase 5

The S3 CSI Driver (Mountpoint for Amazon S3) enables Kubernetes pods to mount S3 buckets as file systems. Used for archive storage.

Why S3 Storage May Be Needed

  • Cost Effective: S3 is significantly cheaper than EBS/EFS for large datasets
  • Scalability: Unlimited storage capacity
