✅ VERSION TAGGING STRATEGY

  All images tagged with v1 (initial release)
  Each component can be updated independently:

  zlaws-backend versions:     v1 → v1.1 (patch) → v2 (major)
  zlaws-postgres versions:    v1 → v1.1 (patch) → v2 (major)
  zlaws-nginx versions:       v1 → v1.1 (patch) → v2 (major)

  Update Process:
    1. Edit component
    2. Build with new version tag: docker build -t zlaws-component:v2 .
    3. Update Kubernetes manifest with new image tag
    4. Deploy: kubectl set image deployment/... component=zlaws-component:v2
    5. Document in IMAGE_VERSION_TRACKER.md
